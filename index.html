CARGO COMMODITY
STANDARDIZATION

20,847 cargo records across three operational systems carried inconsistent, multilingual descriptions — blocking accurate customs classification, volume reporting, and client analytics.

Business Problem

Commodity descriptions were entered as free text by different operators across systems, in French, Arabic, and English — with abbreviations, typos, and no standard. Classification for customs, volume reporting by commodity type, and client-level analytics were all manual and unreliable. One analyst was spending ~4 days per month re-classifying records by hand.

Data Sources

Three source systems: terminal operating system (TOS) manifest exports, Excel-based commercial tracking sheets, and a legacy cargo declaration database. Total: 20,847 rows, 47 columns. 40.4% of commodity description fields flagged as inconsistent on initial audit.

Methodology
1
Text Normalization
Unicode NFKD decomposition, diacritics removal, lowercase, special character stripping
2
Regex Preprocessing
Standardized abbreviations (CONT→CONTAINER), unit normalization, container code extraction
3
Fuzzy Matching
rapidfuzz token_sort_ratio against master taxonomy — 85% threshold, best-match assignment
4
Rules Engine
Keyword dictionary fallback for edge cases not caught by fuzzy logic
Technical Implementation — Python Pipeline
cargo_standardizer.py · Python 3.11
# Stage 1: Text normalization
import unicodedata, re, pandas as pd
from rapidfuzz import fuzz, process

def normalize(text: str) -> str:
    text = unicodedata.normalize('NFKD', str(text))
    text = ''.join(c for c in text if not unicodedata.combining(c))
    text = re.sub(r'[^\w\s]', ' ', text.lower().strip())
    return re.sub(r'\s+', ' ', text)

# Stage 3: Fuzzy match against master taxonomy
def classify(desc: str, taxonomy: dict, threshold=85) -> str:
    norm = normalize(desc)
    match, score, _ = process.extractOne(
        norm, taxonomy.keys(),
        scorer=fuzz.token_sort_ratio
    )
    if score >= 85:
        return taxonomy[match]
    # Stage 4: Keyword fallback rules engine
    for kw, cat in KEYWORD_RULES.items():
        if kw in norm:
            return cat
    return 'UNCLASSIFIED'

    CALCULATE(

TEU Growth YoY =
    DIVIDE(
        [Total TEU] - CALCULATE([Total TEU], SAMEPERIODLASTYEAR(Calendar[Date])),

Avg Dwell Days =
    AVERAGEX(
        FILTER(Containers, Containers[DwellDays] > 0

        Containers[DwellDays]
    )

// 4. Client TEU share — % of total in current period

    DIVIDE(
        [Total TEU],
        CALCULATE([Total TEU], ALL(DIM_Client)),
        0
    )

Business Problem

Inspectors walked the yard with paper forms. Readings were recorded manually, consolidated into an Excel sheet, then emailed to shipping lines — a process that took 2–3 hours from anomaly detection to client notification. Every hour of delay increased cargo damage risk and contractual liability.

System Architecture
1
Power Apps (Tablet)
Inspectors log readings directly from yard. Dropdown-controlled temperature entry with container lookup.
2
SharePoint List (Storage)
Real-time central data store. Each inspection creates a timestamped record with container ID, temperature, and inspector ID.
3
Power Automate (Alert Engine)
Flow monitors SharePoint every 5 min. If temperature > threshold for container type, automated email fires to shipping line + CS team.
4
Power BI (Supervisor Dashboard)
Live reefer yard overview: status map, alert history, container-level detail, daily inspection coverage rate.

MEDITERRANEAN
PORT THROUGHPUT
ANALYSIS

An independent analysis of container throughput divergence between Morocco and Algeria using publicly verified data — Tanger Med official activity reports and World Bank / UNCTAD national statistics. Built entirely outside of any employer context.

Analytical Questions
  • What is Morocco's Tanger Med throughput trajectory and how does it compare to Algeria's national total?
  • What structural differences explain the volume gap — hub strategy vs gateway model?
  • When did Algeria's throughput peak and what drove the post-2018 decline?
  • What are the commercial implications for operators in the region?
Data Sources
  • Tanger Med Port Authority — Official annual activity reports (2019–2024, public)
  • World Bank IS.SHP.GOOD.TU — Container port traffic by country (Algeria series to 2020)
  • UNCTAD / CEIC — Container Port Throughput series, Algeria peak (2018: 1,844,422 TEU)
  • All sources public · data gaps acknowledged explicitly
Technical Stack
  • Python — pandas, matplotlib, seaborn, scipy (correlation)
  • Data cleaning — handling gaps in UNCTAD reporting years, country name normalization across datasets
  • Analysis — CAGR calculation, peer benchmarking, growth decomposition
  • Output — Analytical report + charts

Data note: World Bank container throughput series for Algeria runs to 2020 (1,541,273 TEU). Post-2020 Algerian national data is not consolidated in public international databases as of this analysis. Morocco figures are from the official Tanger Med Port Authority annual activity reports (7,596,845 TEU in 2022; 7,196,000 in 2021).

Key structural finding: Morocco's Tanger Med operates as a transshipment hub on the East–West global route (Strait of Gibraltar), handling cargo that never enters Morocco — a fundamentally different model from Algeria's purely gateway ports. Algeria's throughput reflects only domestic import/export demand. The gap is therefore not purely a competitive loss, but a strategic difference with significant commercial implications for regional positioning.
